# 🎬 Zero-Cost Video Pipeline

> *«Мы не платим за качество. Мы его добираем.»*

Генерация видео на обычном процессоре (CPU). Без видеокарт. Без облака. Без платных сервисов.

## 💡 Идея
Вместо того чтобы генерировать каждый кадр с нуля (дорого), мы:
1. Создаём один ключевой кадр (дёшево, 360p, 4 шага LCM)
2. «Оживляем» только то, что движется (экономия 70-90% вычислений)
3. Улучшаем качество после генерации (селективный апскейл, интерполяция)

## 🚀 Быстрый старт (Dry-Run)
```bash
git clone https://github.com/ВАШ_НИК/zero-cost-video-pipeline.git
cd zero-cost-video-pipeline
chmod +x quickstart.sh
bash quickstart.sh
```
> ⚠️ Сейчас доступен только режим **Dry-Run** (проверка архитектуры). Реальная генерация будет подключена поэтапно. Откройте в браузере: `http://localhost:7860`

## 🧪 Тестирование
1. Откройте `examples/prompts.md` и выберите промпт с меткой `[CPU-Friendly]` или `[Static]`.
2. Вставьте его в интерфейс.
3. Запустите в режиме `Dry-Run`.
4. В логе проверьте:
   - ✅ Кэш: `HIT` / `MISS`
   - ✅ Модели: сколько загружено
   - ✅ Движение: сколько зон определено
   - ✅ Финал: успешная сборка

## 🛡️ Этика и границы
Этот проект создан для **этичной, прозрачной и ресурсно-эффективной** генерации контента.
- ✅ Разрешено: образовательные материалы, арт, маркетинг, исследования
- ❌ Запрещено: спам, манипуляции, вредоносный контент, обход модерации

Подробности: [ETHICS.md](ETHICS.md)

## 🤝 Участие в проекте
Мы принимаем:
- Оптимизацию CPU-инференса (ONNX, OpenVINO, квантование)
- Улучшение кэширования и поиска
- Документацию, примеры, переводы
- Баг-фиксы и упрощение сборки

Не принимаем:
- Зависимости от GPU или платных API
- Скрипты для спама или накрутки
- Изменение этических границ
